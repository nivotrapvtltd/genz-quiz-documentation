Welcome

About Genz Quiz

Genz Quiz is a Laravel 12 quiz platform built for Admins, Teachers, and Students. It includes quiz management, question banks, user roles, paid quizzes, AI-powered question generation, and a modern Blade, Tailwind CSS, and Alpine.js frontend.

Good to know: This documentation is written for non-technical buyers as well as developers. Follow the installer steps in order and do not skip the database setup step.
Before You Start

Server Requirements

  • PHP 8.2 or higher
  • MySQL 8+ or MariaDB equivalent
  • PDO and PDO MySQL extensions enabled
  • OpenSSL, Mbstring, Tokenizer, XML, Ctype, JSON, Fileinfo enabled
  • Apache or Nginx with URL rewriting support
  • Writable storage and bootstrap/cache folders

For shared hosting, make sure your hosting provider supports modern Laravel applications and lets you point your domain to the correct public folder.

Files

Package Contents

Your download package should contain the full Laravel application source code. Important folders include:

app/

Main application logic, controllers, models, services, and middleware.

database/

Migrations and seeders for creating tables and demo content.

public/

Public entry files, compiled assets, and web-facing index file.

resources/

Blade views, styles, and JavaScript source files.

Fast Setup

Quick Start Installation

  1. Upload the project files to your server.
  2. Create a MySQL database and database user from your hosting panel.
  3. Point your domain or subdomain to Laravel’s public folder, or use the shared hosting structure described below.
  4. Visit your domain in the browser.
  5. If the application is not installed yet, it will redirect to the installer.
  6. Complete the installer steps: requirements, access, database, and admin setup.
Important: Never leave your .env file publicly accessible. It contains sensitive server and database credentials.
Recommended

Installer Wizard

The built-in installer is the easiest way to set up Genz Quiz on a fresh server.

Step 1: Requirements Check

The installer checks your PHP version, required extensions, writable directories, and environment file access.

Step 2: Installation Access

This step controls whether installation is open, mock, or Envato-verified depending on the current build configuration.

Step 3: Database Setup

Enter your database host, port, database name, username, and password. The installer tests the connection before moving forward.

Step 4: Application and Admin Setup

Set your site name, application URL, support email, and first administrator login details.

Shared Hosting

cPanel / Shared Hosting Setup

If your hosting uses public_html, the safest setup is to keep the full Laravel application outside the web root and place only the contents of the public folder inside public_html.

Recommended structure

/home/username/genz-quiz
/home/username/genz-quiz/app
/home/username/genz-quiz/bootstrap
/home/username/genz-quiz/config
/home/username/genz-quiz/database
/home/username/genz-quiz/resources
/home/username/genz-quiz/routes
/home/username/genz-quiz/storage
/home/username/genz-quiz/vendor
/home/username/public_html/index.php
/home/username/public_html/.htaccess

If everything must stay inside public_html

You may keep the full project in public_html, but this is less secure. If you do this, protect hidden files and make sure requests route to Laravel correctly.

<Files .env>
    Order allow,deny
    Deny from all
</Files>

<FilesMatch "^\\.">
    Order allow,deny
    Deny from all
</FilesMatch>
Advanced

Manual Setup

If you prefer not to use the installer, you can set up the app manually from SSH or your terminal.

composer install --no-dev --optimize-autoloader
cp .env.example .env
php artisan key:generate
php artisan migrate --seed
php artisan storage:link
php artisan optimize:clear

After manual setup, make sure your .env values are correct and the app URL matches your domain.

Security

File Permissions

The following directories must be writable by the web server:

  • storage/
  • bootstrap/cache/

If permissions are too restrictive, the installer may fail to save configuration or create cache files.

Performance

Queue and Cron Setup

For production use, configure a cron job and queue worker if you plan to use background jobs or scheduled tasks.

* * * * * php /path-to-project/artisan schedule:run >/dev/null 2>&1

If your hosting allows long-running workers, you can also run:

php artisan queue:work --tries=3
Monetization

Stripe / Payment Setup

For premium quizzes, add your payment gateway credentials from the admin settings or through environment variables before going live.

  • Add your Stripe publishable key and secret key
  • Enable HTTPS on your domain
  • Test with a sandbox or test card before accepting live payments
AI Feature

AI Question Generator

The AI question generator helps teachers create quiz questions from a topic prompt. To use real AI generation:

  • Add your OpenAI API key in the application settings or environment
  • Choose the preferred AI model if that option is available in the build
  • Test the generated questions and review them before publishing

If no API key is configured, the platform may use fallback sample questions depending on the application settings.

After Install

Admin Access and First Login

After installation, log in with the administrator email and password you created in the installer.

  • Open the dashboard and review platform settings
  • Configure Web Config for logo, favicon, and homepage content
  • Create categories, quiz posts, and questions
  • Test a student quiz attempt before launch
Help

Troubleshooting

The site shows a database table does not exist error

Your migrations have not run yet. Complete the installer or run php artisan migrate.

The installer keeps redirecting or looks incomplete

Clear Laravel caches with php artisan optimize:clear and make sure APP_INSTALLED is not incorrectly set to true.

The domain opens a 404 or custom hosting error page

Your domain is probably not pointing to Laravel’s public entry point correctly. Re-check your public folder structure and rewrite rules.

Images or uploaded avatars are not showing

Create the storage symlink with php artisan storage:link if your hosting supports it.

Need Help?

Support

  • PHP version
  • Hosting type (shared hosting, VPS, cPanel, etc.)
  • Error message or screenshot

This helps support resolve your issue much faster.
