Welcome

Everything needed to install and integrate Genz Quiz

Genz Quiz is a Laravel 12 quiz platform for Admins, Teachers, and Students. It supports quiz publishing, question banks, premium quiz payments, mobile-friendly student play, and a reusable JSON API for websites, mobile apps, and third-party integrations.

Built For

Admin dashboards, mobile apps, web frontends, and partner integrations.

API Style

JSON-first endpoints with bearer authentication, pagination metadata, and centralized errors.

Access Levels

Public, authenticated user, teacher or admin, and admin-only route groups.

Recommended workflow: Finish installation first, then test login with POST /api/auth/login, then build out quiz, payment, and admin integrations.
Before You Start

Server Requirements

  • PHP 8.2 or higher
  • MySQL 8+ or MariaDB equivalent
  • PDO and PDO MySQL extensions enabled
  • OpenSSL, Mbstring, Tokenizer, XML, Ctype, JSON, Fileinfo enabled
  • Apache or Nginx with URL rewriting support
  • Writable storage and bootstrap/cache folders
Fast Setup

Quick Start Installation

  1. Upload the project files to your server.
  2. Create a database and database user.
  3. Point your domain to the Laravel public directory.
  4. Open the installer in the browser and complete the setup.
  5. Sign in as admin and configure platform, payment, and AI settings.
Important: After deployment or updates, run php artisan optimize:clear so routes, config, and compiled views stay in sync.
Advanced

Manual Setup

composer install --no-dev --optimize-autoloader
cp .env.example .env
php artisan key:generate
php artisan migrate --seed
php artisan storage:link
php artisan optimize:clear

API authentication depends on all migrations being present, including the api_tokens table migration.

Security

File Permissions

  • storage/ must be writable
  • bootstrap/cache/ must be writable
  • public/storage should be a symlink to storage/app/public

If uploads are not visible, verify the storage link from admin settings or run php artisan storage:link.

JSON API

API Overview

The platform exposes a full JSON API for authentication, quiz browsing, quiz play, result history, payments, and admin management. The API is designed to be secure, scalable, and reusable across web and mobile clients.

Base URL

https://your-domain.com/api

Auth Format

Authorization: Bearer YOUR_TOKEN

Content Type

application/json for most endpoints, multipart for avatar upload.

Rate Limiting

General API limits apply to all routes, with stricter limits on auth endpoints.

Recommended headers: Send Accept: application/json on every request. Use Content-Type: application/json for JSON body requests. 
Accept: application/json
Content-Type: application/json
Authorization: Bearer YOUR_TOKEN
Group Who Can Access Examples
Public Anyone /api/ping, /api/meta, /api/quizzes
Auth Anyone for login or register, authenticated users for profile or logout /api/auth/login, /api/auth/me
Student Authenticated users /api/dashboard, /api/my/attempts, /api/quizzes/{quiz}/submit
Payments Authenticated users /api/payments/history, /api/payments/{quiz}/checkout
Admin Teacher or Admin, with some routes Admin-only /api/admin/quizzes, /api/admin/users
Authentication

Auth API

POST /api/auth/login

Authenticate a user and receive a bearer token. This is the main login endpoint for web apps, mobile apps, and third-party clients.

Request

{
  "email": "user@example.com",
  "password": "password"
}

Success Response

{
  "message": "Login successful.",
  "token": "plain-text-api-token",
  "token_type": "Bearer",
  "user": {
    "id": 1,
    "name": "Admin",
    "email": "admin@example.com",
    "phone": null,
    "avatar_url": "https://your-domain.com/storage/avatars/example.png",
    "role": {
      "id": 1,
      "name": "Admin",
      "slug": "admin"
    },
    "is_active": true
  }
}
Required fields: email, password Optional fields: device_name Content type: application/json or application/x-www-form-urlencoded
GET /api/auth/login

Returns endpoint usage help when opened directly in the browser.

{
  "message": "Use POST to log in and receive a bearer token.",
  "endpoint": "/api/auth/login",
  "method": "POST",
  "content_type": "application/json or application/x-www-form-urlencoded",
  "required_fields": ["email", "password"],
  "optional_fields": ["device_name"]
}
POST /api/auth/register

Create a student account and receive a bearer token immediately.

{
  "name": "Student User",
  "email": "student@example.com",
  "password": "Password123!",
  "password_confirmation": "Password123!",
  "phone": "9876543210",
  "device_name": "android-app"
}
{
  "message": "Registration successful.",
  "token": "plain-text-api-token",
  "token_type": "Bearer",
  "user": {
    "id": 12,
    "name": "Student User",
    "email": "student@example.com",
    "role": {
      "id": 3,
      "name": "Student",
      "slug": "student"
    }
  }
}
GET /api/auth/register

Returns browser-safe registration instructions and required fields.

GET /api/auth/me

Returns the authenticated user profile.

PUT /api/auth/me

Updates profile fields including name, email, phone, avatar, and password.

POST /api/auth/logout

Deletes the current API token and logs the user out.

Reliability

Errors and Security

404 JSON Errors

Unknown API routes return JSON instead of an HTML error page.

405 Browser Safety

Opening auth endpoints directly in the browser returns usage instructions instead of raw method errors.

429 Rate Limiting

API and auth routes are rate limited centrally to reduce abuse and brute-force login attempts.

Secure Headers

API responses include hardened headers such as X-Frame-Options, X-Content-Type-Options, and cache restrictions for auth-sensitive responses.

Common error responses

{
  "message": "The provided credentials do not match our records."
}
{
  "message": "An unexpected API error occurred.",
  "status": 500
}
Troubleshooting

Frequently Asked Questions

Why does login return a 500 error?

Check pending migrations, clear Laravel caches with php artisan optimize:clear, and make sure the running server uses the same database as your terminal.

Why do uploaded avatars not load?

Make sure public/storage is a valid symlink to storage/app/public.

Why do auth endpoints show JSON in the browser?

That is intentional. Browser-safe help responses prevent confusing 405 HTML pages for API-only routes.

How should mobile apps store tokens?

Store bearer tokens securely in the platform's secure storage, then send them in the Authorization header on protected requests.

Need Help

Support Notes

When requesting support, include the endpoint, HTTP method, request body, response body, and the exact timestamp of the issue. That makes server log checks much faster.
